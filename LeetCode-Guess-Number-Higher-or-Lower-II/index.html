LeetCode 375 - Guess Number Higher or Lower II

Posted on 2021-05-23 In LeetCode Views: Disqus:

題目

題目連結：https://leetcode.com/problems/guess-number-higher-or-lower-ii/

猜數字遊戲，答案在 1∼n1 \sim n1∼n 之內。

每次猜測一個數字 xxx，如果猜對了則遊戲結束，若猜錯了則需要付 xxx 元，並且會被告之正確答案大於 xxx 或是小於 xxx。

找出一種猜數字的方式，使得不管答案是多少，花費都是最少的（也就是不管答案是多少，花費最大的那種答案要盡量小）。

想法

首先，以遞迴的想法來說。一開始要求的是猜測 1∼n1 \sim n1∼n 的數字的最小花費，定為 minCost(1, n)。

假設猜測的數字為 xxx，且 1≤x≤n1\le x \le n1≤x≤n，則有三種情形：

  1. 猜中了，則花費為 0，遊戲結束。
  2. 沒猜中，且正確答案大於 xxx，則花費 xxx 元且繼續猜測 x+1∼nx+1 \sim nx+1∼n。
  3. 沒猜中，且正確答案小於 xxx，則花費 xxx 元且繼續猜測 1∼x−11 \sim x-11∼x−1。

由於三種情況都是有可能發生的，因此：

minCost(1,n)=max⁡{0case 1x+minCost(x+1,n)case 2x+minCost(1,x−1)case 3minCost(1,n)=\max \begin{cases} 0 &\text{case 1} \\ x+minCost(x+1,n) &\text{case 2} \\ x+minCost(1,x-1) &\text{case 3} \\ \end{cases} minCost(1,n)=max⎩⎪⎨⎪⎧​0x+minCost(x+1,n)x+minCost(1,x−1)​case 1case 2case 3​

有了上述的式子，我們可以總結並整理，對於猜測 i∼ji \sim ji∼j 之間的數字的最小花費，定為 minCost(i, j)，我們可以猜測任何一個 i≤k≤ji\le k \le ji≤k≤j 的數字 kkk，因此：

實作細節

實作上，當然可以直接的使用遞迴來求出答案，但是會發現很多情況下重複的 minCost(i, j) 被呼叫，但其實對於一樣的 minCost(i, j)，並不需要重複的求得。

因此，可以記憶化搜索的方式，也就是開一個二維的陣列 dp[i][j] 代表 minCost(i, j) 的答案。初始化時使全部的 dp[i][j] = -1，則呼叫 minCost(i, j) 時可以先檢查 dp[i][j]，若 dp[i][j] != -1 則直接回傳 dp[i][j]。

有了這樣的想法後，其實可以很輕易的將 top-down 的遞迴改為 botton-up 的迴圈形式 DP；改為迴圈形式之 DP 時，要注意迴圈的順序，由於大區間的答案是由小區間求得的，因此要先求得小區間答案，一般情況下，都可以使用以下的迴圈順序：

此時的 DP 狀態為 dp(i, j) 代表猜測數字 i∼ji \sim ji∼j 的最小花費，轉移為：

dp(i,j)=min⁡(max⁡(dp(i,k−1),dp(k+1,i))+k∀ k∈[i,j])dp(i,j)=\min\bigg(\max\big(dp(i,k-1), dp(k+1,i)\big)+k\quad\forall\ k\in [i,j]\bigg) dp(i,j)=min(max(dp(i,k−1),dp(k+1,i))+k∀ k∈[i,j])

注意邊界情況，當 i=ji=ji=j，一定會猜中，因此 dp(i,i)=1dp(i,i)=1dp(i,i)=1。

最後，DP 的狀態有 N2N^2N2 種，對於每個狀態需要花至多 O(N)O(N)O(N) 的時間轉移，因此總時間複雜度為 O(N3)O(N^3)O(N3)。

另外，由於筆者的陣列只有開恰好 nnn 格，因此要注意 dp(i,j) 其實是對應到 i+1∼j+1i+1\sim j+1i+1∼j+1 的最小花費。
